SRC配置

如您是第一次使用SRC，SRC可能会返回 NullStudentNemesList 错误，这是因为你没有向SRC提供学生姓名
SRC将会从 src/src-config/student_names.txt 中读取学生姓名，你应该在编辑此文件以提供它

打开此文件，你将会看到以下提示信息



不必担心提示信息会对学生姓名的读取尝试影响，以 # 开头的行（必须是第一个字符）将会被SRC忽略， 你可以使用 # 暂时注释掉某个名字。
正如提示信息所提示的，SRC以空格和换行分隔学生姓名，此处不再赘述
值得一提的时，如果你使用Windows notepad（记事本）等编辑器编辑此文件并保存，可能会使一个不可见的数据插入到文件中， 导致文件第一行的 # 失效，此行提示信息被当作姓名处理

SRC使用

src中提供了3个版本的exe可执行文件，分别是src/SRC/SRC.exe，src/SRC-GUI/SRC-GUI.exe， src/SRC-GUIlit/SRCGUIlit.exe，你应自行为其创建快捷方式

SRC-GUI

src/SRC-GUI/SRC-GUI.exe
SRC的GUI界面，允许你打开命令行、打开文档、随机不放回抽取学生姓名

在2025.11.13的更新中，新增了一个按钮，打开SRC GitHub上的项目地址，方便你下载更新的版本

SRC-GUIlit

src/SRC-GUIlit/SRCGUIlit.exe
轻量化的GUI界面，没有SRC-GUI的主界面和所有命令行功能，界面如SRC-GUI的抽取界面

SRC command-line

src/SRC/SRC.exe
SRC的命令行界面，提供了GUI界面的一切功能，同时提供了一些高级功能，请查看SRC command-line文档

SRC记忆功能

在2025.10.28的更新中，默认在随机不放回抽取时减去上次抽取的姓名，以实现记住你上次抽取剩下姓名的功能。如你需要重新抽取， 你可以：
手动清空文件src/src-history/last_chioced_names.txt，里面保存了你上次抽取的姓名，下次抽取时将会减去其中的姓名。 清空文件，下次打开将重新开始抽取
修改src/src-config/student_names.txt，当SRC发现此文件遭到有效更改时，将会清空副本和上次抽取姓名，重新抽取。 这里只是提醒你修改你提供的学生信息会使SRC重新抽取

SRC日志

在2025.11.13的更新中，新增了日志文件src/src-history/src-log.txt，这里面保存了SRC每次打开的时间、抽取姓名、 修改副本和已抽取姓名文件的行为
